# 🧠 我的 PARA 知识库

## 什么是 PARA 方法？

PARA 是 **Tiago Forte** 提出的知识管理框架，将信息组织为四个类别：

| 类别 | 英文 | 定义 | 特点 |
|------|------|------|------|
| **项目** | Projects | 有明确目标和截止日期的短期努力 | 临时性、可完成 |
| **领域** | Areas | 需要长期维护的生活领域 | 持续性、无明确结束 |
| **资源** | Resources | 感兴趣的主题或参考材料 | 可复用、随时查阅 |
| **归档** | Archives | 已完成或不活跃的内容 | 备查、历史记录 |

## 🚀 快速开始

### 1. 创建新项目
1. 进入 `00_Projects/相应分类/`
2. 使用模板：[[Templates/项目模板]]
3. 命名格式：`项目名_简短描述`

### 2. 记录技术学习
1. 进入 `01_Areas/技术学习/具体领域/`
2. 使用模板：[[Templates/技术笔记模板]]
3. 添加相关标签和双向链接

### 3. 收集资源
1. 进入 `02_Resources/相应分类/`
2. 创建笔记或收藏链接
3. 定期回顾和整理

### 4. 归档项目
1. 项目完成后，移动到 `03_Archives/`
2. 添加复盘笔记
3. 保持链接完整性

## 📝 使用规范

### 命名规范
- **项目**：`项目名_简短描述`（例：`React项目_组件库重构`）
- **笔记**：使用清晰、可搜索的标题
- **日期格式**：`YYYY-MM-DD`

### 标签系统
- `#project` - 项目笔记
- `#learning` - 学习笔记
- `#active` - 正在进行中
- `#review` - 需要复盘
- `#todo` - 待办事项

### 链接规范
- 使用双向链接 `[[笔记名]]` 建立关联
- 在相关笔记间创建上下文链接
- 维护 MOC（内容地图）进行主题汇总
