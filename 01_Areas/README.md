# 📂 Areas - 领域

## 定义
**领域**是需要持续投入和长期维护的生活领域，没有明确的截止日期。

## 特点
- 🔄 持续进行，没有明确结束
- 📊 需要定期维护和改进
- 🎯 有标准和目标
- 📈 长期跟踪

## 子分类

### 📁 职业发展
- 工作管理
- 技术成长路径
- 行业洞察
- 职业规划

### 📁 技术学习
- 前端开发
- 后端开发
- 通用技能

### 📁 生活管理
- 健康管理
- 财务规划
- 人际关系
- 个人成长

### 📁 兴趣探索
- 阅读
- 旅行
- 其他爱好

## 使用规范

### 与项目的区别
| 项目 (Projects) | 领域 (Areas) |
|----------------|--------------|
| 短期、有截止日期 | 长期、持续维护 |
| 完成后归档 | 持续更新 |
| 具体目标 | 持续改进 |

### 维护频率
- 每周回顾一次
- 每月更新目标和标准
- 定期整理和优化

### 内容组织
每个领域应该包含：
1. **MOC（内容地图）** - 领域知识框架
2. **目标与标准** - 期望达到的水平
3. **现状记录** - 当前状态
4. **改进计划** - 如何提升
