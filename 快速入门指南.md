# 🚀 快速入门指南

欢迎使用 PARA 方法 Obsidian 知识库！这份指南将帮助你在 10 分钟内上手使用。

---

## 📋 目录结构总览

```
Obsidian Vault/
├── 00_Projects/          ← 短期项目（有截止日期）
├── 01_Areas/             ← 长期领域（持续维护）
│   └── 技术学习/         ← 前端/后端/通用技能
├── 02_Resources/         ← 参考资料、收藏
├── 03_Archives/          ← 已完成归档
├── Templates/            ← 5种笔记模板
├── Dashboard.md          ← 你的主页
└── README.md             ← 使用说明
```

---

## 🎯 第一步：设置主页

1. 打开 **Dashboard.md**
2. 将其设为 Obsidian 的启动页：
   - 设置 → 核心插件 → 启用「Daily notes」
   - 或使用 Commander 插件设置快捷键
3. 每次打开 Obsidian 都会看到仪表盘

---

## 📝 第二步：创建你的第一个项目

### 方法 A：使用模板（推荐）
1. 进入 `00_Projects/学习项目/`
2. 新建笔记，命名为：`你的项目名`
3. 使用模板：
   - 安装 **Templater** 插件
   - 按 `Ctrl/Cmd + P` → 输入 "Template" → 选择「项目模板」
4. 填写项目信息

### 方法 B：手动创建
1. 复制 `Templates/项目模板.md` 的内容
2. 粘贴到新笔记中
3. 修改内容

### 💡 示例
我已经为你创建了一个示例：[[00_Projects/学习项目/React进阶_30天精通|React进阶项目]]

## 🔄 第四步：每日复盘

### 创建每日复盘
1. 进入 `01_Areas/生活管理/`
2. 新建文件夹 `每日复盘/`
3. 每天创建新笔记，使用「每日复盘模板」
4. 或直接在 Dashboard 中点击今日复盘链接

### 快速访问
在 Dashboard 中已添加：[[01_Areas/生活管理/每日复盘/{{date:YYYY-MM-DD}} 复盘|📝 开始记录]]

## 🔗 第五步：建立链接

### 双向链接
在 Obsidian 中，使用 `[[笔记名]]` 创建链接：
```markdown
我在学习 [[React 18 新特性]] 时遇到了问题，
需要参考 [[React Hooks]] 的文档。
```

### 查看关系图
- 按 `Ctrl/Cmd + G` 打开图谱视图
- 可以看到笔记之间的连接关系

## 📊 推荐插件

为了获得最佳体验，建议安装以下插件：

### 必需插件
1. **Dataview** - 动态查询和列表
2. **Templater** - 模板自动化
3. **Calendar** - 日历视图（配合 Daily notes）

### 推荐插件
4. **Kanban** - 看板管理项目
5. **Mind Map** - 思维导图
6. **Tag Wrangler** - 标签管理
7. **Periodic Notes** - 周期性笔记

### 安装方法
1. 设置 → 第三方插件 → 关闭安全模式
2. 浏览社区插件
3. 搜索并安装

## 💡 使用技巧

### 1. 快捷键
- `Ctrl/Cmd + E` - 切换编辑/预览模式
- `Ctrl/Cmd + O` - 快速打开文件
- `Ctrl/Cmd + Shift + F` - 全局搜索
- `Ctrl/Cmd + G` - 打开图谱

### 2. 命令面板
按 `Ctrl/Cmd + P` 打开命令面板，快速执行操作：
- 插入模板
- 新建笔记
- 切换视图

### 3. 快速捕获
使用「待处理」文件夹作为收件箱：
1. 遇到有价值的内容，先放到 `02_Resources/待处理/`
2. 每周固定时间整理
3. 归档到相应分类

## ❓ 常见问题

**Q: 如何决定内容放在哪个文件夹？**
A: 使用 PARA 决策树：
- 有截止日期？→ Projects
- 长期维护？→ Areas
- 参考资料？→ Resources
- 已完成？→ Archives

**Q: 模板中的变量不生效怎么办？**
A: 需要安装并启用 **Templater** 插件，并正确配置模板文件夹。

**Q: Dataview 查询不显示结果？**
A: 确保：
1. 已安装 Dataview 插件
2. 笔记中有正确的 frontmatter（头部元数据）
3. 查询语法正确
